Welcome to the How-To Guide

This guide will cover the basics of using Petfinder's API to create a pet adoption website. We will be going over what an API is, what sort of data the API will send, and how to extract data from Petfinder's database. Althought we will be going over several aspects of the API in detail, it is assumed that you have a basic understanding of HTML, CSS, JavaScript, and DOM.

What is an API?

There are many kinds of APIs, but we will be talking specifically about web APIs. API stands for application programming interface. Merriam-Webster defines an interface as “the place at which independent and often unrelated systems meet and act on or communicate with each other”. An API is simply an interface for two unrelated software applications to communicate with each other. In our case, the Petfinder API will allow our website to interact with Petfinder’s database of adoptable pets and extract the desired information to be displayed on our website.


Imagine an API is like a menu from a restaurant.

The customer (our website) looks through the information presented on the menu (the API). When the customer decides what they want to order, they will give the server their order who will relay the message to the chef (Petfinder). The chef cooks up the order, gives it to the server, who then presents you with your meal (the desired data). The meal is now yours to eat, give to another person, spill on the table, or anything you can think of -- as long as you're not breaking any terms of agreement.
