Data Format

Next we will be discussing the format of the data being sent. Petfinder supports two formats: XML and JSON. XML is the default format, however, we will be working with JSON. It is very important to understand the structure of JSON because all of the information we request will be wrapped up in JSON and it requires specific operators to extract data. We will be discussing the basics of JSON, but a much more thorough explanation of JSON can be found here.

JSON

JSON stands for JavaScript Object Notation. Data exchanged between servers can only be in text. JSON is in an all text format, but is structured like a JavaScript object. This allows us to utilize the dot operator when extracting information. These two examples contain the same information except one is in JSON format and the other is a JavaScript object.

JavaScript JSON

The data is written as name/value pairs. As you can see from the two examples, the main difference between JSON and JavaScript is that the name portion of the pairs in JSON have to be in quotes while JavaScript does not. It is also important to note that the assigned value can be an object. In our example, we have a an object named pizza with two name/value pairs: Hawaiin/object and Vegetarian/object. These objects are also in JSON format whose name/value pairs are: name/(name of pizza) and toppings/(array of toppings). The values in JSON can be of most types including a string, a number, a JSON object, an array, a boolean, or null. However, it cannot be a function, a date, or undefined.
