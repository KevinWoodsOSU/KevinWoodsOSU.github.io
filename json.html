Data Format

Next we will be discussing the format of the data being sent. Petfinder supports two formats: XML and JSON. XML is the default format, however, we will be working with JSON. It is very important to understand the structure of JSON because all of the information we request will be sent as a JSON string and it requires specific operators to extract data. We will be discussing the basics of JSON, but a much more thorough explanation of JSON can be found here.

JSON

JSON stands for JavaScript Object Notation. Data exchanged between servers can only be in text. JSON is an all text format, but is structured like a JavaScript object. This allows us to utilize the dot operator when extracting information by easily switching from JSON to JavaScript object and vice-versa. These two examples contain the same information except one is in JSON format and the other is a JavaScript object.


JavaScript JSON

The data is written as name/value pairs. As you can see from the two examples, the main difference between JSON and JavaScript is that the name portion of the pairs in JSON have to be enclosed in double quotes while a JavaScript object does not. It is also important to note that the assigned value can be a nested object in JSON format. In our example, we have a an object named pizza with two name/value pairs: Hawaiin/object and Vegetarian/object. These objects are also in JSON format whose name/value pairs are: name/(name of pizza) and toppings/(array of toppings). The values in JSON can be of most types including a string, a number, a JSON object, an array, a boolean, or null. However, it cannot be a function, a date, or undefined like with JavaScript objects.

Accessing Data from JSON

As mentioned above, due to the similar structure between JSON and JavaScript objects, we can use dot notation to access specific data from objects of JSON format. In our above example, we use pizza.Vegetarian.toppings[0] to grab the value "olives". This table illustrates how to cascade down an objects of JSON format until you reach your desired value that is of a simple data type like a string.

One thing to remember when using dot notation is if there is a name with a space in your object (ex. New York Style) then you will need to use bracket notation to access its value. Accessing New York Style toppings would look like pizza["New York Style"].toppings[0].

The last bit about handling JSON data is converting JavaScript to JSON and vice-versa. JavaScript has two built-in functions for handling these operations: JSON.stringify converts a JavaScript object to a JSON string and JSON.parse converts a JSON string to a JavaScript object.
